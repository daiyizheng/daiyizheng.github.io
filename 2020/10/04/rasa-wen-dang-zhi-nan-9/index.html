部署Rasa助手

本页说明何时以及如何部署使用Rasa构建的助手。它将使您的助手可供用户使用，并为您准备好生产环境。

何时部署您的助手

部署助手并使之可供测试用户使用的最佳时间是可以处理最重要的快乐路径，或者是我们所谓的最低可行助手。

通过以下推荐的部署方法，可以通过Rasa X中的共享助手功能轻松地与测试用户共享助手。然后，当您准备通过一个或多个消息和语音通道使您的助手可用时，您可以轻松地将它们添加到现有的部署设置中。

推荐部署方法

建议的部署助手的方法是使用我们支持的“服务器快速安装”或“头盔图表”选项。两者都部署Rasa X和您的助手。它们是部署助手，允许您使用Rasa X来查看对话并将其转换为培训数据的最简单方法，并且已准备就绪。有关部署方法的更多详细信息，请参见《Rasa X安装指南》。

服务器快速安装

服务器快速安装脚本是部署Rasa X和您的助手的最简单方法。它使用合理的默认值在您的机器上安装了Kubernetes集群，使您可以通过一条命令启动并运行。

  • 默认值：确保满足操作系统要求，然后运行：

    curl -s get-rasa-x.rasa.com | 须藤bash

    复制
    复制！

  • 自定义：请参阅自定义脚本 和服务器快速安装文档。

舵图

对于将吸引大量用户流量的助手，通过我们的Helm图表设置Kubernetes或Openshift部署是最佳选择。这提供了可扩展的体系结构，该体系结构也易于部署。但是，如果有特殊要求，您也可以自定义Helm图表。

  • 默认值：阅读Helm Chart安装文档。
  • 自定义：阅读以上内容以及“高级配置” 文档，然后根据需要自定义开源Helm图表。

替代部署方法

Docker

您也可以在没有集群环境的情况下在Docker Compose设置中运行RasaX。我们有一个这样做的安装脚本，以及任何自定义设置的手动说明。

  • 默认值：阅读Docker Compose安装脚本文档或观看有关部署Rasa X的Masterclass视频。
  • 自定义：阅读Docker Compose手动安装文档以获取完整的自定义选项。

Rasa仅限开源部署

也可以使用Docker Compose在不使用Rasa X的情况下部署Rasa助手。为此，您可以在本地或在Docker中构建Rasa Assistant。然后，您可以在Docker Compose中部署模型。

部署动作服务器

建设行动服务器映像

如果您构建包含操作代码的映像并将其存储在容器注册表中，则可以将其作为部署的一部分运行，而不必在服务器之间移动代码。此外，您可以添加系统或Python库的任何其他依赖关系，这些依赖关系是动作代码的一部分，但不包含在基本rasa/rasa-sdk映像中。

创建图像：

  1. 将操作代码移到actions项目目录中的文件夹中。确保还添加一个空actions/__init__.py文件：

    mkdir actions
mv actions.py actions/actions.py
touch actions/__init__.py  # the init file indicates actions.py is a python module

    该rasa/rasa-sdk图像会自动寻找在行动actions/actions.py。

  2. 如果您的操作有任何其他依赖性，请在文件中创建一个列表 actions/requirements-actions.txt。

  3. Dockerfile在项目目录中创建一个名为的文件，您将在其中扩展官方SDK映像，在代码上进行复制并添加任何自定义依赖项（如果需要）。例如：

    ＃扩展官方Rasa SDK映像
FROM rasa/rasa-sdk:1.10.2

＃使用子目录作为工作目录
WORKDIR /app

＃如有必要，复制任何其他自定义要求（取消注释下一行
# COPY actions/requirements-actions.txt ./

＃更改回root用户以安装依赖项
USER root

＃如有必要，安装操作代码的其他要求（取消注释下一行）
# RUN pip install -r requirements-actions.txt

＃将动作文件夹复制到工作目录
COPY ./actions /app/actions

＃按照最佳做法，请勿以root用户身份运行代码
USER 1001

然后，您可以通过以下命令生成映像：

docker build . -t <account_username>/<repository_name>:<custom_image_tag>

该<custom_image_tag>应参考这一形象将如何与其他人不同。例如，您可以标记标签的版本或日期，以及为生产和开发服务器创建具有不同代码的不同标签。每当您更新代码并想重新部署它时，都应该创建一个新标记。

使用自定义动作服务器映像

如果要构建此映像以使其可从其他服务器（例如Rasa X或Rasa Enterprise部署）使用，则应将映像推送到云存储库。

本文档假定您将映像推送到DockerHub。DockerHub将让您免费托管多个公共存储库和一个私有存储库。确保首先创建一个帐户 并创建一个存储库来存储您的图像。您还可以将映像推送到其他Docker注册表，例如Google Container Registry， Amazon Elastic Container Registry或 Azure Container Registry。

您可以通过以下方式将映像推送到DockerHub：

docker login --username <account_username> --password <account_password>
docker push <account_username>/<repository_name>:<custom_image_tag>

要将图像进行身份验证并将其推送到其他容器注册表，请参阅所选容器注册表的文档。

您如何引用自定义操作映像取决于您的部署。选择与您的部署相关的文档：
